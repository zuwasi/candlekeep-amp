Follow-up: Nuanced Scope Comparison

If MISRA Rule 11.3 passes, can I skip CERT EXP36-C?
Not exactly. MISRA 11.3 is broader (bans ALL pointer type conversions), but CERT EXP36-C catches void* round-trip alignment issues outside MISRA's scope. Mostly yes, but CERT adds edge cases.

Install in 2 Minutes

Works on Windows, macOS, and Linux.

Download the package

Get the latest release from GitHub:

# Download from GitHub Releases page
→ github.com/zuwasi/candlekeep-amp/releases

Run the installer (Windows)

PS> powershell -ExecutionPolicy Bypass -File install.ps1

Installs ck.exe, sets up the Amp skill, and opens browser for authentication.

Upload your standards

$ ck items add MISRA-C-2023.pdf
$ ck items add CERT-C-2016.pdf
$ ck items add AUTOSAR-CPP14.pdf

Ask questions in Amp

You: "Cross-reference MISRA Rule 11.3 across all my standards"
You: "What does my library say about pointer casting?"
You: "Which of my standards cover null pointer dereferencing?"

Manual Installation

# 1. Install ck CLI
$ cargo install candlekeep-cli   # or download binary from GitHub Releases

# 2. Authenticate
$ ck auth login

# 3. Copy skill files to Amp skills directory
$ cp -r skills/candlekeep ~/.agents/skills/candlekeep
